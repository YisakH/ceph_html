Interface for block encryption methods

Crypto filters for Put/Post/Get operations.

Encrypts and decrypts data. Operations are performed in context of larger stream being divided into blocks. Each block can be processed independently, but only as a whole. Part block cannot be properly processed. Each request must start on block-aligned offset. Each request should have length that is multiply of block size. Request with unaligned length is only acceptable for last part of stream.

Decrypts data. Argument stream_offset shows where in generalized stream chunk is located. Input for decryption is input buffer, with relevant data in range <in_ofs, in_ofs+size). input and \output may not be the same buffer.

\params input - source buffer of data in_ofs - offset of chunk inside input size - size of chunk, must be chunk-aligned unless last part is processed output - destination buffer to encrypt to stream_offset - location of <in_ofs,in_ofs+size) chunk in data stream, must be chunk-aligned

반환값
true iff successfully encrypted
